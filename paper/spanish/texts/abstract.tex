\begin{abstract}

\textit{Propósito:} Crear arquetipos openEHR a partir de definiciones de recursos FHIR que permitan importar datos de recursos FHIR dentro de sistemas openEHR.

\textit{Métodos:} Un prerrequisito para convertir instancias de recursos FHIR en instancias de arquetipos openEHR es encontrar equivalencias entre los tipos de datos FHIR y openEHR. En este trabajo, se propone las condiciones para establecer dichas equivalencias. Adicionalmente, se presenta una forma de crear arquetipos openEHR utilizando las equivalencias.

\textit{Resultados:} Las correspondencias entre los tipos de datos de FHIR y openEHR. Con base en estas equivalencias, se diseño un proceso de creación automático basado en un sistema de tipos con soporte para vinculaciones a terminologías.

\textit{Conclusiones:} El proceso automático posibilita la creación de arquetipos openEHR de recursos FHIR de forma eficiente, y se asigna como responsables de la importación de datos a los especialistas de la tecnología de información. La compatibilidad entre los tipos de datos de FHIR y openEHR garantiza que se pueda convertir y procesar correctamente los datos de instancias de recursos FHIR en instancias de arquetipos openEHR.

\end{abstract}
